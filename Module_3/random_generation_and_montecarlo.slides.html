Random Numbers Generation¶

Random number generator¶

Random number generators (RNGs) are devices or programs that produce unpredictable and unbiased sequences of numbers.

Why generate random numbers?¶

Random numbers can be used to:

  • simulate different outcomes of an experiment
  • generate secure keys to encrypt and decrypt data
  • model different market scenarios, estimating the probability of occurrence of specific outcomes

How can random numbers be generated?¶

  • hardware-based generators - use physical phenomena such as thermal noise, radioactive decay, or quantum effects to generate random numbers
  • software-based generators - algorithms that rely on mathematical equations to generate a sequence of numbers

Good generators¶

A working definition of randomness in the context of computer-generated sequences is the following:

Any two different random number generators ought to produce statistically the same results when coupled to your particular applications program. If they don’t, then at least one of them is not a good generator.

The random modules¶

There are several methods implemented in different languages to generate random numbers:

  • RandomNumberGenerator class in C++ - provides many functions for generating random values, providing also various distributions such as normal, exponential, gamma
  • NativeCrypto library in Android and iOS - provides cryptographically secure random values
  • the TrueRandom Arduino library - create random numbers using external sources such atmospheric noise, or internal sources like ADC readings

Among the standard python libraries, we have access to 3 set of random functions generation:

  • random
  • numpy.random
  • scipy.stats

The linear congruential generator¶

A very simple way of generating random numbers (not a good one, but a good example) is the following:

$$R_{n+1} = (a \cdot R_n + c) \text{mod} m$$

where a is a positive integer called the multiplier, and c is a nonnegative integer called the increment.

For $c \neq 0$, the above equation is called a linear congruential generator (LCG).

Weakness of LCGs¶

If k random numbers at a time are used to plot points in k-dimensional space (with each coordinate between 0 and 1), then the points will not tend to “fill up” the k-dimensional space, but rather will lie on (k-1)-dimensional “planes.” There will be at most about $m^{1/k}$ such planes. If the constants m and a are not very carefully chosen, there will be many fewer than that.

Inverse transform method¶

Inverse transform sampling is a method for generating random numbers from any probability distribution by using its inverse cumulative distribution $F^{−1}(x)$.

The cumulative distribution for a random variable $X$ is $F_{X}(x)=P(X≤x)$.

Algorithm¶

We want to generate a random variable $X$ with cumulative distribution function (CDF) $F_{X}$.

The algorithm is simple:

  • generate a sequence of random numbers $R$ between 0 and 1 drawn from a uniform distribution
  • find $X=F^{-1}(R)$

For the exponential distribution, can write the following functions:

def exp_pdf(x, b=1):
    return b*np.exp(-b*x)

def exp_cdf(x, b=1):
    return 1 - np.exp(-b*x)

def exp_inv_cdf(p, b=1):
    return -np.log(1-p)/b

What if we have data with unkown distribution?¶

Even if we have data with unkonwn distribution, we can still use the inverse transform method to create a random number generator.

In this case, the inverse CDF function can be estimated using interpolation.

def compute_ecdf(data):
    data_sorted = np.sort(data)
    index = np.linspace(0, 1, 1+len(data_sorted))[1:]
    return data_sorted, index

Monte Carlo methods¶

A collection of methods that uses random sequences to estimate the average value of some specific functions, often integrals.

An example is the estimation of the value of $\pi$ using random numbers

$$ \int_{[0,1]^s} f(u)\,{\rm d}u \approx \frac{1}{N}\,\sum_{i=1}^N f(x_i). $$

The rate of convergence by using N random numbers is $O\left(\frac{1}{\sqrt{N}}\right)$.

The rate of convergence using a uniform sampling of N numbers is $O\left(\frac{1}{N}\right)$.

Sampling using random numbers converge more slowly, but can be performed for as long as we want until we reach the convergence up to a certain precision.

With uniformely sampled numbers, if the chosen sampling wasn't enough, we have to start from scratch and redo everything.

Low-discrepancy sequences¶

There are better methods for exploring a function if we can, and that is by using low-discrepancy sequences.

These are sequences that look quite random, but do not tend to cluster as random numbers do, covering the space in a more uniform way.

The big advantage is that they keep having this property if we continue forward with the sequence of sampling.

Low-discrepancy sequences converge with $O\left(\frac{1}{N}\right)$, but they are a little more difficult to implement.

These methods are called quasi-Monte Carlo, as they are not using proper pseudo-random numbers

Van der Corput sequence¶

The simplest low-discrepancy sequence is the Van der Corput sequence, and is quite easy to implement.

The denominators are increasing powers of the base, the numerators are all the values in that base, but sorted based on the last digit (skipping the numbers that ends in 0, as they would simplify and recreate previous ones)

$$\left\{ \tfrac{1}{10}, \tfrac{2}{10}, \tfrac{3}{10}, \tfrac{4}{10}, \tfrac{5}{10}, \tfrac{6}{10}, \tfrac{7}{10}, \tfrac{8}{10}, \tfrac{9}{10}, \tfrac{01}{100}, \tfrac{11}{100}, \tfrac{21}{100}, \tfrac{31}{100}, \tfrac{41}{100}, \tfrac{51}{100}, \tfrac{61}{100}, \tfrac{71}{100}, \tfrac{81}{100}, \tfrac{91}{100}, \tfrac{02}{100}, \tfrac{12}{100}, \tfrac{22}{100}, \tfrac{32}{100}, \ldots \right\}$$
In [750]:
def vdc(n, base=2):
    vdc = 0
    denom = 1
    while n:
        denom *= base
        n, remainder = divmod(n, base)
        vdc += remainder / denom
    return vdc

def avdn(n, base=10):
    return np.array([vdc(i, base=base) for i in range(1, n+1)])

A general library to generate these sequences in high dimensions is provided by the library SALib.

You won't need it for today, but for real use, where the number of dimensions is high, it might be worth using it.
